# CodexPlusPlus 定制与 Windows EXE 交付计划

## 总体目标

在现有 Rust + Tauri 2 + React 架构上完成开机自启、仓库链接迁移、更新源迁移、TUN 检测反转、推荐内容本地化、赞赏图替换、普通供应商压缩百分比设置，以及按模型独立路由的“添加供应商(可自定义)”功能 Meow~

最终版本统一提升到 `1.2.36`，交付两个原始 EXE 和一个 Windows x64 安装程序 Meow~

## 接口与数据模型

- 在 [settings.rs](D:/Cursor/CodexPP/crates/codex-plus-core/src/settings.rs) 中为 `RelayMode` 增加 `CustomModels`，JSON 值使用 `customModels` Meow~
- 新增 `CustomRelayModel`，字段固定为内部 `id`、`model`、`baseUrl`、`apiKey`、`protocol`、`contextWindow`、`autoCompactEnabled` 和 `autoCompactPercent` Meow~
- `RelayProfile` 新增 `customModels: Vec<CustomRelayModel>`、`defaultCustomModelId`、`autoCompactEnabled` 和 `autoCompactPercent`，保留 `autoCompactLimit` 作为旧配置兼容字段 Meow~
- `autoCompactPercent` 合法范围为 `1..=100`，默认显示 `80`，开关默认关闭，避免升级后擅自改变现有供应商行为 Meow~
- 新增 `AutostartStatus { supported, enabled, executablePath }`，以及 Tauri 命令 `get_manager_autostart` 和 `set_manager_autostart_enabled` Meow~
- 自定义供应商显式模型名采用严格匹配，显式传入未知模型时拒绝请求；请求未携带模型时使用 `defaultCustomModelId` 对应模型 Meow~

## 构建与交付

- 全程只使用 Windows PowerShell、Cargo、npm 和原生 Windows 工具，禁止使用 WSL Meow~
- 先在 `D:\Cursor\CodexPP\apps\codex-plus-manager` 执行 `npm ci`、`npm run check` 和 `npm run build`，该命令会构建前端、launcher 和 Tauri manager Meow~
- 原始可执行文件必须出现在 `D:\Cursor\CodexPP\target\release\codex-plus-plus.exe` 和 `D:\Cursor\CodexPP\target\release\codex-plus-plus-manager.exe` Meow~
- 本机当前缺少 NSIS，需先安装原生 Windows NSIS，并确认 `makensis.exe` 可调用 Meow~
- 将两个 EXE 复制到 `D:\Cursor\CodexPP\dist\windows\app\`，再执行现有 `scripts\installer\windows\CodexPlusPlus.nsi`，传入 `/DVERSION=1.2.36` Meow~
- 最终安装程序明确输出为 `D:\Cursor\CodexPP\dist\windows\CodexPlusPlus-1.2.36-windows-x64-setup.exe` Meow~
- 对三个 EXE 生成 SHA-256，并实际启动 manager、验证托盘、推荐图片和自定义模型配置后再交付 Meow~
- 新仓库当前还没有 Release，因此本地 EXE 可正常交付，但自动更新检查要等首次发布 `v1.2.36` 且工作流上传 `latest.json` 后才会返回有效结果 Meow~
